gitCommand

git命令稍作整理(总是忘记，干脆系统的整理一下吧)

  • git初始化
  • 版本回退
  • 远程仓库

初始化git仓库：

进入到该目录下，使用命令 git init 会产生一个.git文件夹

添加文件到git仓库

使用命令 git add <file>添加文件
使用命令 git commit -m "说明文字" 提交文件
git status 查看状态

版本回退

git log 显示从最近到最远的提交日志
git reset --hard HEAD~1 回到上一个版本,
    HEAD 表示当前版本，HEAD~2表示回到上上个版本，依次类推
git reset --hard <ef5e463 版本号>
    回到后一个版本，这个时候需要版本号
git reflog 记录我们的每一次命令，其中有版本号

撤销修改

1.若是修改了没有add: 直接 git checkout -- <file> 注意--后面有一个空格
2.若是修改了也add了：先git reset HEAD <file>从暂存区中撤回
                    然后 git checkout -- <file> 撤销修改
3.若是已经commit了，考虑进行版本回退那一步

删除文件

本地删除提交之后的文件之后
1.若是误删，可以恢复：git checkout -- test.txt
2.若是想彻底删除： git rm test.txt 
                    git commit -m "remove test.txt"

关联远程仓库：首先在官网new responsitory

git remote add origin git@server-name:path/repo-name.git
git push -u origin master第一次推送master分支所有内容
后面的push只需要git push origin master命令

从远程克隆

git clone https://github.com/Echoting/learngit.git

创建与合并分支

查看分支 git branch
创建分支 git branch <name>
切换分支 git checkout <name>
创建+切换分支 git checkout -b <name>
合并指定分支到当前分支 git merge <name>
删除分支 git branch -d <name>

解决冲突

合并的时候如果有冲突，就手动修改，然后提交
git log --graph命令可以看到分支合并图
