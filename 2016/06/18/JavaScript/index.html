javascript 初级的一点点总结（上）

最近有关于JavaScript看后的一点心得体会（上）

  • 自定义属性，索引值的使用
  • JavaScript的基本的数据类型

  • 作用域问题

    索引值的使用

    1）自定义属性：JS 可以为任何HTML元素添加任意个自定义属性 

    aBtn[0].abc = 123;       // 自定义属性

    注：不能对背景进行判断，颜色不能判断，and相对路径不能判断
    2）自定义属性：索引值得使用
    想建立“匹配”“对应”关系，就用索引值

    var arr = [ '莫涛', '张森', '杜鹏' ];    
for( var i=0; i<aBtn.length; i++ ){
    aBtn[i].index = i;            // 自定义属性（索引值）    
    aBtn[i].onclick = function (){
        // alert( arr[ this.index ] );
        this.value = arr[ this.index ];        
        aP[ this.index ].innerHTML = arr[ this.index ];
    };
}

    基本的数据类型及typeof

    1）基本的数据类型
    ECMAScript：标准、核心
    HTML 标签类型：block、inline、inline-block、table……
    JS中的数据类型：数字、字符串、布尔、函数、对象(obj、[]、{}、null)、未定义
    2）typeof 判断数据类型（string number object function undefined）
    3）isNaN： Is not a number 是不是不是一个数字（不是数字）
    isNaN(a):就是对a进行Number(a)操作，如果返回值为NaN，则返回1，否则为0.

    作用域的问题

    变量分为局部变量和全局变量，分别有不同的作用域范围。

    JS解析器：

    1）找一些东西：var function 参数 【预解析的过程】 

    a = ...

    所有的变量，在正式运行代码之前，都提前赋了一个值：未定义 

    fn1 = function fn1(){ alert(2); }

    所有的函数，在正式运行代码之前，都是整个函数块
    注：遇到重命名的，只留一个，如果是变量和函数重命名了，就只留下函数。
    注意：火狐不能对如下函数进行预解析，会报错，尽量避免这种写法

    alert( fn1 );        // FF 不能对下面的函数进行预解析
if( true ){
    var a = 1;
    function fn1(){
       alert(123);
    }
}

    2）逐行解读代码
    表达式：= + - * / % ++ – ! 参数……
    表达式可以修改预解析的值！

    alert(a);                    // function a (){ alert(4); }
var a = 1;
alert(a);                    // 1
function a (){ alert(2); }
alert(a);                    // 1
var a = 3;        
alert(a);                    // 3
function a (){ alert(4); }
alert(a);                    // 3

想要获取函数内的值

方法1. 讲函数内的值赋值给一个全局变量

var str = '';
function fn1(){
    var a = '大鸡腿~';
    str = a;
}
fn1();

方法2： 在函数内再调用一个函数，将里面的值“偷”出来

function fn2(){
    var a = '9999999克拉钻石23456789';
    fn3(a);
}
fn2();
function fn3(a){
    alert(a);
}
